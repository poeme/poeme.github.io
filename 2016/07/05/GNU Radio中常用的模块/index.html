GNU Radio中常用的模块

通过学习 Doxygen（文档生成器）生成的说明文档来了解 block 是个好方法。

假设你已经安装了 Doxygen，那么在安装 gnuradio-core 和 usrp packages 时文档就
已经生成了。它们在如下路径中：
/usr/local/share/doc/gnuradio-core-x.xcvs/html/index.html
/usr/local/share/doc/usrp-x.xcvs/html/index.html

block是由C++编写的。例如，在FM接收机例子中 (/gnuradiocore\src/python/gnuradio/blks2impl/wfm_rcv.py)
，我们使用的block
gr.quadrature_demod_cf。这个block相当于C++编写的类
gr_quadrature_demod_cf。SWIG提供了它到Python的接口。SWIG做了一些工
作，以致从Python的角度来看，block变成了在模块gr中定义的类
quadrature_demod_cf，这样我们就可以在python中，以 gr.quadrature_demod_cf
来接入这个 block了。查看Doxygen生成的文档时，前缀gr, qa, usrp 在 Python
中相当于模块名，前缀之后的部分是模块中block的名称，例如
quadrature_demod_cf, fir_filter_fff。所以如果编写一个block 为’A_BC…’，那
在Python中就称为’A.BC…’。

GNURadio中，所有的信号处理模块都是采用C++编写，采用SWIG提供python和C++之间的接口，因此python可以直接调用C++模块。
GNURadio的C++模块可以在路径 < top gr
dir>/gnuradio-core/src/lib/中找到。

  • filter-数字滤波器模块
  • general-数据类型转换模块（e.g. float to complex），大部分特殊运算的信号处理模块（e.g. frequency recovery,gain control,etc.）。
  • genen-单一的数字运算，比如加，乘。
  • io-输入输出模块，包括文件和矢量。
  • runtime-GNURadio的核心功能，内部工作函数。
